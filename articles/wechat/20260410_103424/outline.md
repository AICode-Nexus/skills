# Outline

## Hook

从“写需求、画原型”走向“直接做可验证产品”，越来越多非研发角色开始需要一个真正能跑起来的软件起点，而不是一张图或一页静态稿。

## Main Sections

### 1. 现在最大的瓶颈，已经不是想法，而是初始化

- Depth levers: failure mode, comparison, decision rule
- Explain the gap between a prompt-generated page and a maintainable project
- Compare static mockup, one-off page, and waiting for engineering bandwidth
- Practical takeaway: project initialization is the new bottleneck for AI-assisted product building

### 2. `react-monorepo-init` 到底一次性帮你搭了什么

- Depth levers: mechanism, implementation detail, boundary
- Walk through the actual stack and why each layer exists
- Explain optional add-ons and why the scope is intentionally constrained
- Practical takeaway: the value is not feature completeness; the value is a professional baseline

### 3. 最能打动人的部分，是跑完之后的项目结构

- Depth levers: implementation detail, comparison, implication
- Show a curated repo tree code block
- Explain why the structure appeals to non-engineers and frontend engineers differently
- Practical takeaway: “看起来专业” is not cosmetic; it changes collaboration confidence

### 4. 为什么前端同学也应该关心这种 skill

- Depth levers: tradeoff, mechanism, counterexample
- Clarify this is not replacing frontend engineers
- Explain standardization, lower bring-up cost, and improved AI stability
- Practical takeaway: use it to eliminate repetitive setup, not to erase engineering judgment

### 5. 为什么最先受益的反而是产品、设计、运营、销售

- Depth levers: concrete scenarios, decision rule, operational implication
- Role-by-role examples
- Clarify what each role can now do differently with AI
- Practical takeaway: a new working pattern for MVPs and internal tools

### 6. 现在就可以怎么开始

- Depth levers: actionability, example, precise commands
- Include npm and GitHub links
- Include installation commands for Codex, Claude Code, and project-local usage
- Practical takeaway: lowest-friction path to try this week

## Closing

AI coding 真正变成生产力，不在于一句 prompt 能出多少代码，而在于它从什么样的标准起点开始工作。把起点标准化，才有资格谈规模化复用。
