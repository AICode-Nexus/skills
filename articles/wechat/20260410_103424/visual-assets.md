# Visual Assets Notes

## Asset 1 - Cover Image

- Placement: article header
- Goal: immediately communicate "non-frontend people can launch a professional React project"
- Delivered files:
  - `cover.svg`
  - `cover.png`
- Direction: dark technical cover with a clean project-tree / monorepo composition
- Key text:
  - Main: `不会前端，也能起专业 React 项目`
  - Sub: ``react-monorepo-init` for demo / MVP / internal tools`
- Visual elements:
  - one highlighted `apps/web` card
  - several secondary package cards: `ui`, `api-client`, `theme`, `hooks`
  - subtle code/editor framing, not marketing-style icons
- Tone: credible engineering aesthetic, not exaggerated AI sci-fi

## Asset 2 - Project Structure Inline Figure

- Placement: immediately before or after the repo tree code block in section three
- Goal: reinforce that the generated result is professional, not toy scaffolding
- Delivered files:
  - `project-structure-figure.svg`
  - `project-structure-figure.png`
- Format options:
  1. screenshot-style code panel showing the curated tree
  2. simplified diagram with `apps/web` and `packages/*`
- Caption suggestion:
  - `跑完之后，你拿到的不是一个页面，而是一套像样的前端项目。`

## Asset 3 - Role Benefit Diagram

- Placement: before section five
- Goal: help non-technical readers map the skill to their own work
- Structure:
  - left: roles `产品 / 设计 / 运营 / 销售`
  - center: `react-monorepo-init + AI agent`
  - right: `Demo / MVP / Internal Tool / Customer PoC`
- Caption suggestion:
  - `不同角色拿到的不是同一种价值，但都共享同一个专业起点。`

## Usage Notes

- If only one image is used, prioritize Asset 1.
- If two images are used, prioritize Asset 1 + Asset 2.
- Asset 2 is the strongest proof-oriented visual for frontend readers.
- Current recommendation:
  - use `cover.png` as the WeChat cover
  - keep `project-structure-figure.png` inline in section three
